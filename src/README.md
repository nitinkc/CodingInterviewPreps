# Coding Interview Preparation - Source Code Organization

## Overview
This directory contains a **linear and incremental learning path** for Algorithm and Data Structure preparation, organized into 4 main categories.

## Directory Structure

### 📚 Learning Path (in order)

#### 1. **01-fundamentals/** - Foundation Concepts
Start here to build the fundamental knowledge required for algorithms and data structures.
- **a0bitwise/** - Bitwise operations and bit manipulation techniques
- **a1primitives/** - Primitive data types, integer/character/string operations
- **a1DataStructures/** - Introduction to basic data structure concepts
- **a2utilities/** - Common utility classes and helper functions

**Learning focus:** Understand how computers work at the bit level, master primitive data types, and build utility functions you'll use throughout.

---

#### 2. **02-data-structures/** - Core Data Structures
Master fundamental and advanced data structures used in every algorithm.
- **arrays/** - Array operations and array-based problems
- **ArraysProblems/** - Specific array manipulation problems
- **linkedLists/** - Linked list implementations and variations
- **stacks/** - Stack data structure and applications
- **binarySearchTrees/** - BST fundamentals and operations
- **binaryTrees/** - Tree traversals, modifications, and problem-solving
- **heaps/** - Priority queues and heap operations
- **tries/** - Prefix trees for string operations
- **graphs/** - Graph representations and basic operations
- **a3DataStructures/** - Advanced data structure concepts
- **CircularQueue.java** - Specialized queue implementation

**Learning focus:** Understand how data is organized, stored, and accessed. These are the building blocks for all algorithms.

---

#### 3. **03-algorithms/** - Core Algorithms
Learn the algorithms that solve common computational problems efficiently.
- **sorting/** - Sorting algorithms (bubble, insertion, merge, quick, etc.)
- **searching/** - Search techniques (binary search, etc.)
- **recursion/** - Recursive problem-solving patterns
- **DynamicProgramming/** - DP principles, memoization, and optimization
- **famousAlgorithms/** - Classic algorithms (Dijkstra, Floyd-Warshall, etc.)
- **a4Algorithms/** - Additional algorithmic techniques

**Learning focus:** Apply data structures to solve problems efficiently. Master common patterns like sorting, searching, recursion, and dynamic programming.

---

#### 4. **04-practice/** - Interview Problems & Practice
Apply what you've learned by solving real interview problems from multiple sources.
- **algoEx/** - AlgoExpert problems (well-structured, comprehensive)
- **leetCode/** - LeetCode problems (diverse difficulty levels)
- **crackingTheCoding/** - "Cracking the Coding Interview" problems
- **skiena/** - Algorithm Design Manual problems
- **solutionProjectEuler/** - Project Euler mathematical problems
- **topCoder/** - TopCoder competitive programming problems
- **udemyJobInterview/** - Udemy interview preparation course
- **manage/** - Problem management and tracking utilities
- **strings/** - String manipulation problems

**Learning focus:** Synthesize knowledge by solving diverse problems. Focus on understanding multiple solutions and approaches to the same problem.

---

## Recommended Learning Path

### Phase 1: Foundation (Weeks 1-2)
1. Study **01-fundamentals/a0bitwise/** - Bit operations
2. Study **01-fundamentals/a1primitives/** - Primitive types
3. Study **01-fundamentals/a1DataStructures/** - Basic concepts

### Phase 2: Data Structures (Weeks 3-6)
1. **arrays** → understand memory layout
2. **linkedLists** → pointers and references
3. **stacks** + **heaps** → queue-like structures
4. **binarySearchTrees** + **binaryTrees** → hierarchical data
5. **graphs** → relational data
6. **tries** → string-based trees

### Phase 3: Algorithms (Weeks 7-10)
1. **sorting/** - Foundation for many algorithms
2. **searching/** - Optimization techniques
3. **recursion/** - Problem decomposition
4. **DynamicProgramming/** - Optimization patterns
5. **famousAlgorithms/** - Real-world applications

### Phase 4: Practice (Weeks 11+)
1. Start with **algoEx** (well-structured learning)
2. Progress to **leetCode** (broad coverage)
3. Use **crackingTheCoding** for interviews
4. Challenge yourself with **topCoder** for competitive programming

## Key Principles

✅ **Linear Learning** - Start with foundations, build incrementally  
✅ **Organized by Concept** - Not by source or difficulty  
✅ **Reduced Complexity** - Clear categorization instead of 30+ folders  
✅ **Practice Integration** - Apply knowledge through problems  
✅ **No Code Changes** - Only reorganization (comments/logic preserved)
